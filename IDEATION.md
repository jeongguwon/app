# 스쿨줍줍 기획서

학교 구성원 전용 분실물·습득물 매칭 PWA.
3개 모델(Claude Opus 4.7 · Gemini 2.5 Pro · GPT-5.5) 교차검증으로 다듬은 최종안.

---

## 1. 개요

| 항목 | 내용 |
|---|---|
| 프로젝트명 | 스쿨줍줍 |
| 한 줄 소개 | 학교 구성원 전용 분실물·습득물 매칭 PWA |
| 대상 | 단일 학교의 학생·교직원(인증된 사람만) |
| 플랫폼 | 모바일 우선 반응형 웹앱(PWA) |
| 운영 주체(예) | 학생회 또는 학교 행정실 |

## 2. 문제와 해결책

**문제** — 학교 공식 인스타그램은 공지·분실물이 섞이고 검색이 불가능해 게시물이 빠르게 묻힌다. 운영자만 글을 올릴 수 있어 등록도 지연된다.

**해결책** — 분실물 전용 공간에서 카테고리·위치·날짜로 검색하고, 등록자 본인이 상태를 관리하며, 비공개 식별 질문으로 진짜 주인만 연락 채널을 얻는다.

## 3. 사용자와 권한

| 역할 | 권한 |
|---|---|
| 일반(학생·교직원) | 등록·검색·연락·신고·내 글 상태 변경 |
| 관리자(학생회·행정실) | 신고 처리, 게시글 숨김/삭제, 카테고리·장소 관리 |

## 4. 인증

학교 이메일 도메인(@school.ac.kr 등) OTP. 외부 도메인 차단. 14세 미만은 법정대리인 동의 절차.

## 5. 데이터 (게시글)

**공통 필드** — 종류(분실/습득), 사진(최대 4장), 제목, 카테고리, 색상, 장소(사전 정의 목록), 일시, 설명, 상태.

**습득물 전용** — 보관 장소(행정실/경비실/도서관 데스크/개인 보관), 비공개 식별 질문.

**분실물 전용** — 비공개 식별 정보(소유 증명용).

## 6. 상태 흐름

```
등록됨 → 연락 중 → 반환 완료 → 30일 후 자동 비공개 → 90일 후 영구 삭제
```

작성자 또는 관리자만 상태를 바꿀 수 있고, 자동 만료는 시스템 배치가 처리한다.

## 7. 핵심 흐름 — 양방향 소유권 검증

| 경로 | 시작점 | 검증 방법 |
|---|---|---|
| A | 분실자 → 습득물 글 | "내 물건 같아요" → 습득자 질문에 답변 |
| B | 습득자 → 분실물 글 | "이거 제가 주웠어요" → 분실자 식별 정보 확인 |

- 검증 통과 시 연락 채널 오픈
- 양쪽 "수령 확인" 클릭 시 `반환 완료`
- 식별 질문 3회 연속 실패 시 24h 잠금 + 작성자에게 "여러 명 시도 중" 알림

## 8. 연락 방식

전화번호·카톡 ID 직접 노출 금지. 검증 통과 후 "연락 요청"을 작성자가 수락하면 앱 내 메시지 1회 전달. (v2에서 인앱 1:1 채팅으로 확장.)

## 9. 화면

| 화면 | 구성 |
|---|---|
| 홈 | 검색바, 카테고리 칩, 최근 등록 카드 리스트(반응형 1~2열), 우하단 (+) 등록 |
| 리스트 | 검색 + 필터(카테고리·장소·기간·상태) |
| 상세 | 사진 갤러리, 정보, 액션("내 물건 같아요"/"이거 제가 주웠어요"/신고/작성자 상태 변경) |
| 등록 | 사진 → 종류 → 카테고리 → 제목 → 색상 → 장소 → 일시 → 설명 (+습득물: 보관 장소·비공개 질문) |
| 마이페이지 | 내 글, 진행 중 매칭, 알림함, 내 데이터 삭제 요청 |
| 관리자 콘솔 | 신고 큐, 게시글 관리, 위험물 인계 큐 |

## 10. 알림 (PWA 웹 푸시)

신규 검증 요청 도착 / 검증 결과 / 상태 변경 시 발송.

## 11. 안전·정책

- 사진 업로드 시 EXIF/GPS 메타데이터 제거
- 어뷰징 방지: 게시글 등록 시간당 5건, 식별 질문 글당 3회/24h
- 금지 물품(현금·신분증·의약품 등): 등록 시 "행정실 전달" 경고 모달
- 반환 완료 후 사진 즉시 삭제, 본문 90일 후 영구 삭제
- 개인정보 처리방침·이용약관 페이지
- **출시 전 학교 행정실 검토 필수**

## 12. 카테고리

전자기기 · 지갑/카드 · 가방 · 의류/신발 · 학용품/책 · 액세서리 · 우산 · 열쇠 · 기타

## 13. MVP 범위

1. 학교 이메일 인증
2. 분실·습득 등록 (EXIF 제거)
3. 리스트 + 검색·필터
4. 상세
5. 양방향 소유권 검증
6. 연락 요청
7. 상태 관리
8. 신고
9. 마이페이지
10. 관리자 콘솔(신고·숨김)
11. PWA 푸시

## 14. v2 이후

인앱 1:1 채팅 · 자동 매칭 알림 · 이미지 유사도 매칭 · 다중 학교 지원 · 다국어(영어)

## 15. 기대효과 (정량 지표)

- 인스타 대비 평균 매칭 소요 시간 단축률
- 월간 분실물 회수율
- 학교 공식 계정 분실물 게시 부담 감소율
- 활성 사용자 비율(MAU/가입자)

## 부록 A. 검토 히스토리

원본 아이디어부터 최종안에 도달한 과정 기록.

### A.1 원본 아이디어

- **문제점**: 학교 공식 인스타그램으로 분실물을 찾을 때 게시물을 일일이 찾아야 하고, 공지사항과 섞여 묻히기 쉬움
- **기능**: 학교 맞춤형 / 카테고리별 등록 / 발견자와 연락(소유권 확인)
- **화면 구성**: 상단 로고, 중단 분실물·습득물 등록 버튼, 카드 2열 리스트, 상세 화면 하단 연락 버튼

### A.2 1차 검토에서 도출된 보완점

사용자 인증 / 소유권 검증 / 연락 방식(번호 비노출) / 상태 관리 / 위치 정보 / 검색·필터 / 알림 / 신고·관리 / 법규(유실물법·개인정보보호법·14세 미만 동의).

### A.3 교차검증 결과 — 세 모델이 모두 동의한 항목

| 비공개 식별 질문 | ✅ | ✅ | ✅ |
| 전화번호 직노출 금지 | ✅ | ✅ | ✅ |
| 상태 관리 | ✅ | ✅ | ✅ |
| 검색 + 필터 | ✅ | ✅ | ✅ |

| 유실물법·개인정보보호법 고지 | ✅ | ✅ | ✅ |

**모델별 강조점 차이**
- Gemini: 초기 사용자 확보(닭과 달걀) 문제, 인앱 채팅 강력 권장
- GPT-5.5: 습득물은 학교 공식 보관처에 맡기고 앱은 조회·매칭만 — 유실물법 관점에서 가장 안전
- Claude: 14세 미만 법정대리인 동의, 사진 속 타인 가림 안내

### A.4 2차 보완 (추가 검토에서 발견)

1. 양방향 매칭 흐름 (분실자→습득물뿐 아니라 습득자→분실물 경로도 필요)
2. 사진 EXIF/GPS 메타데이터 제거
3. 데이터 보관/삭제 기간 명시
4. 플랫폼은 PWA로 결정 (스토어 심사 없음, 단일 코드베이스, 웹 푸시 가능)
